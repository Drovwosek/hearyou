# Задача: перевод HearYou STT на локальный Whisper

## Цель

Полностью перевести сервис транскрибации с Yandex SpeechKit на локальный Whisper-бэкенд.

Yandex STT больше не используется ни как основной, ни как fallback. Проект должен разворачиваться в облаке, но без внешней STT-зависимости.

## Что нужно сделать

1. Добавить локальный STT-бэкенд на базе Whisper.
2. Убрать зависимость от `core/yandex_stt.py` из основного runtime-пути.
3. Сохранить внешний API сервиса без ломающих изменений:
   - `/transcribe`
   - `/status/{task_id}`
   - `/result/{task_id}`
   - `/download/{task_id}`
   - `/history`
   - `/stats`
4. Сохранить текущие возможности пайплайна:
   - предобработка аудио;
   - очистка текста;
   - speaker diarization;
   - обработка больших файлов через текущую очередность/чанки, если это необходимо.
5. Подготовить проект к облачному развёртыванию без Yandex API ключей.

## Ограничения

- Не оставлять Yandex STT как обязательную зависимость.
- Не ломать текущий пользовательский интерфейс и формат ответов без необходимости.
- Не делать миграцию в один монолитный рефакторинг, если можно сохранить совместимость через адаптер.
- Если потребуется выбор модели Whisper, предпочтение отдавать конфигурируемости через env.

## Предлагаемая архитектура

### STT-провайдер

Сделать единый абстрактный интерфейс для распознавания речи, например:

- `transcribe_sync(...)`
- `transcribe_async(...)`
- `check_operation(...)`
- `wait_for_completion(...)`

Реализация по умолчанию должна быть локальной.

### Локальный движок

Рекомендуется один из вариантов:

- `faster-whisper`
- `whisper.cpp`

Выбор должен учитывать:

- запуск в контейнере;
- работу в облаке;
- возможность использовать CPU-only режим;
- предсказуемую память и время обработки.

### Конфигурация

Добавить конфигурируемые параметры через environment variables:

- модель Whisper;
- устройство выполнения (`cpu` / `cuda`, если применимо);
- размер батча;
- язык по умолчанию;
- режим точности/скорости;
- путь к временным файлам.

## Что проверить при реализации

- Сервис стартует без `YANDEX_API_KEY`.
- Тесты проходят без внешних облачных STT-зависимостей.
- Интеграционные маршруты поднимаются на чистом checkout.
- Обработка файла возвращает результат в прежнем формате.
- Диаризация продолжает работать поверх результата Whisper.
- Облачный деплой остаётся возможным через Docker.

## Критерии готовности

- Yandex STT полностью исключён из основного production path.
- Локальный Whisper используется по умолчанию.
- API контракты не сломаны.
- Проект собирается и запускается в облаке без секретов Yandex.
- Есть документация по запуску и выбору модели Whisper.

## Порядок работ

1. Найти все места, где `YandexSTT` участвует в runtime.
2. Вынести STT-слой в отдельный интерфейс/адаптер.
3. Реализовать локальный Whisper backend.
4. Подключить его в `packages/stt-service/app.py`.
5. Обновить зависимости, Dockerfile и документацию.
6. Прогнать тесты и поправить несовместимости.
7. Удалить остатки Yandex-only кода, если они больше не нужны.

## Примечания

- Сохранить speaker diarization как отдельный шаг после распознавания.
- Если локальный Whisper окажется слишком тяжёлым для текущего контейнера, нужно явно зафиксировать требования к CPU/RAM/GPU в документации.
- Если часть старой логики нужна для миграции, она должна быть временной и изолированной.
